## 단계별 핵심 차이

### 1단계 — 현재 POC
```
할 수 있는 것: 숫자 비교 (접근로 < 3m, 나이 >= 80)
못 하는 것:    맥락 판단 ("서울 자녀 연락은 의미없다")
특징:          빠르게 만들 수 있다. 예측 가능하다. 하지만 딱딱하다.
```

### 2단계 — TypeDB 온톨로지
```
할 수 있는 것: 관계 탐색 (마순례 → 건물 → 이웃 → 자원)
못 하는 것:    판단의 무게 결정 ("지금 이 상황에서 뭐가 더 중요한가")
특징:          데이터 연결은 잘 된다. 하지만 추론은 여전히 규칙 의존.
```

### 3단계 — 온톨로지 + LLM
```
할 수 있는 것: 맥락 판단, 우선순위 조정, 예외 상황 대응
못 하는 것:    실시간 현장 감각 (이장이 지금 어디 있나, 불이 어느 방향으로 번지나)
특징:          베테랑 소방관의 판단에 가장 가깝다.
               단, 판단 근거를 반드시 기록해야 책임 소재가 생긴다.
```

## 결론 — 베테랑 소방관이 기준이다

베테랑 소방관의 판단 칸을 보면 패턴이 있다.

1. **기억 기반**: "그 집이 샌드위치 패널이고 골목이 좁다"는 걸 이미 알고 있다
2. **관계 기반**: "마 할머니는 귀 어두우시고 서울 자녀는 멀다"는 연결을 안다
3. **경험 기반**: "2분이면 플래시오버다, 지금 바로 뛰어야 한다"는 감각이 있다
4. **현장 기반**: "바람 방향 보니까 옆집도 위험하다"를 눈으로 판단한다

1~3번은 온톨로지 + LLM으로 구현 가능하다.
4번은 실시간 센서 데이터(풍향, CCTV)가 연결되어야 가능하다.

**PATOS의 목표는 1~3번을 15초 안에 자동화하는 것이다.**
4번은 현장 소방관이 한다.
PATOS는 결정을 대신하는 게 아니라, 판단에 필요한 정보를 15초 안에 정리해주는 것이다.
