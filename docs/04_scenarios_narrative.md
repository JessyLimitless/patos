## 시나리오 A — FIRE-2026-001 `CRITICAL`

### 상황 서술

새벽 2시 15분. 소록마을 안쪽 골목 끝 23-7번지에서 전기 합선으로 불이 났다.
마을 주민 한 명이 연기를 보고 119에 신고한다.

> *"불이야! 소록마을 안쪽 마 노인네 집에 불이 났어요! 빨리요! 거기 어르신 귀가 어두우셔서 모르실 것 같아요!"*

### 데이터값

| 항목 | 값 |
|------|-----|
| 건물 | BLDG-007 · 소록마을 23-7 · 샌드위치 패널 · 2001년 |
| 접근로 | **1.8m** — 소방차 물리적 진입 불가 |
| 화재감지기 | **없음** |
| 가로등 | **없음** |
| 거주자 | 마순례(PERSON-001) · 84세 · 여 · 독거 |
| 신체 조건 | **난청 + 거동불편** + 당뇨 · mobility_score 0.16 |
| 비상연락처 | 마씨 자녀 · **서울** (2~3시간 거리) |
| 기상 | 풍속 7.2m/s · 습도 22% · 기온 11.5°C · **건조** |
| 플래시오버 | **약 2.5분** |

### 시스템이 해야 하는 판단

1. 신고 내용에서 "마 노인네 집", "귀가 어두워서 모르실 것 같아요" 추출 → PERSON-001 특정
2. BLDG-007의 접근로 1.8m → 소방차 진입 불가 확정
3. 풍속 7.2m/s + 습도 22% + 샌드위치 패널 → 플래시오버까지 2.5분, 소방차 ETA와 **9.5분 공백** 계산
4. mobility_score 0.16 + 난청 → 자력 대피 불가, 경보 인지 불가 → 도보 접근 필수
5. **즉시 발령**: 마을방송 + 이장 ARS + 도보 선발대

### 현재 시스템의 한계

- 비상연락처(서울 자녀)에 자동 연락 — **의미 없음**. 서울에서 내려오는 데 2시간 이상.
- "이장이 지금 마을에 있는지" 실시간 위치를 모름
- 풍향 데이터 없음 → 불이 어느 방향으로 번질지 모름 → 인접 BLDG-006(슬레이트 지붕, 접근로 2.0m)의 조만수(81세, 난청) 위험을 감지 못함

### 이 상황에서 LLM이 해야 할 추가 판단

> "비상연락처는 서울이라 실질 도움 불가. 이장 위치 확인 후 도보 선발대로 즉시 파견. 풍속 7.2m/s 고려 시 남서방향 BLDG-006도 5분 내 연소 위험. 마순례 씨는 mobility 0.16으로 들것 이동 필요 — 선발대에 이동 보조 장비 지참 지시."

## 시나리오 B — FIRE-2026-002 `HIGH`

### 상황 서술

새벽 2시 43분. 소록마을 12-3번지 보일러실에서 불이 시작됐다.
지나가던 주민이 연기를 보고 신고한다.

> *"여보세요 119죠? 소록마을 12번지 쪽에서 연기가 나고 있어요. 이 할아버지 혼자 사시는데 치매가 있으셔서 대피를 못 하실 것 같아요."*

### 데이터값

| 항목 | 값 |
|------|-----|
| 건물 | BLDG-003 · 소록마을 12-3 · 경량 목구조 · 1978년 |
| 접근로 | **2.5m** — 소방차 진입 불가(기준 3m 미만) |
| 화재감지기 | **없음** |
| 가로등 | 있음 |
| 거주자 1 | 이봉길(PERSON-003) · 78세 · 남 · 독거 |
| 신체 조건 | **치매 + 난청** + 고혈압 · mobility_score **0.79** |
| 거주자 2 | 김복순(PERSON-002) · 79세 · 여 · 독거 |
| 신체 조건 | **치매** + 고혈압 · mobility_score **0.51** |
| 기상 | 풍속 5.5m/s · 습도 31% · 기온 10.2°C · 흐림 |
| 플래시오버 | **약 4분** |

### 시스템이 해야 하는 판단

1. "이 할아버지 혼자" → 이봉길(PERSON-003) 특정, 하지만 같은 주소에 **김복순(PERSON-002)도 거주** → 2명 동시 위험
2. 접근로 2.5m → 소방차 진입 불가, 도보 분대 필수
3. 플래시오버 4분 → 소방차보다 8분 빠름 → 도보 선발대 선파견
4. **구조 우선순위 판단 필요**: 이봉길(치매+난청, mobility 0.79) vs 김복순(치매만, mobility 0.51)

### 현재 시스템의 한계

- priority_score 공식으로는 이봉길(78세 + 치매 + 난청)이 더 높은 점수 → 먼저 구조
- 하지만 **실제 판단**: 이봉길은 mobility 0.79로 혼자 걸을 수 있음. 김복순은 0.51로 더 느림.
- 보일러실 발화 → **연기는 아래에서 위로 이동**, 보일러실 인접한 방의 거주자가 먼저 위험
- 두 거주자가 **같은 건물 어느 방에 있는지** 모름

### 이 상황에서 LLM이 해야 할 추가 판단

> "두 명 동시 위험. 이봉길은 mobility 0.79이지만 치매+난청으로 지시 불가. 김복순은 mobility 0.51이지만 청력 정상 — 큰 소리로 호출 시 반응 가능성 있음. 선발대 2명 이상 파견 필요. 보일러실 발화이므로 연기는 내부 복도 타고 이동 — 빠른 창문 개방으로 환기 유도."

## 시나리오 C — FIRE-2026-003 `MODERATE`

### 상황 서술

새벽 3시 7분. 소록마을 35-9번지 창고에서 불이 났다.
강 씨 내외가 거주하는 집으로 연기가 들어가고 있다고 신고가 들어온다.

> *"소록마을이에요. 35번지 창고에서 불이 났어요. 연기가 집 안으로 들어가고 있어요. 강 씨 내외분이 계신 것 같아요."*

### 데이터값

| 항목 | 값 |
|------|-----|
| 건물 | BLDG-010 · 소록마을 35-9 · **조적조** · 1988년 |
| 접근로 | **3.5m** — 소방차 진입 가능 |
| 화재감지기 | **있음** (작동 여부 불명) |
| 가로등 | 있음 |
| 거주자 1 | 강덕현(PERSON-008) · 75세 · 남 · **동거** |
| 신체 조건 | 당뇨 + 관절염 · mobility_score 0.78 |
| 거주자 2 | 윤금순(PERSON-009) · 73세 · 여 · **동거** |
| 신체 조건 | 고혈압 + 당뇨 · mobility_score 0.85 |
| 기상 | 풍속 3.2m/s · 습도 45% · 기온 13°C · 맑음 |
| 플래시오버 | **약 11분** (조적조 + 습도 45%) |

### 시스템이 해야 하는 판단

1. 조적조 + 접근로 3.5m → 소방차 진입 가능 → 표준 대응 가능
2. 두 거주자 mobility 0.78 / 0.85 → **자력 대피 가능성 높음**
3. 창고(별채) 발화 → 플래시오버까지 11분, 소방차가 1분 여유 있음
4. 화재감지기 있으나 신고자가 외부에서 발견 → **감지기 미작동** 의심

### 현재 시스템의 한계

- "표준 대응"으로 분류하면 끝 — 하지만 창고→본채 연소 진행 속도 계산이 없음
- 화재감지기가 있는데 왜 안 울렸는지 → 배터리 방전? 고장? 이게 맥락을 바꿈
- 부부 동거이므로 한 명이 이미 다른 한 명을 깨웠을 가능성 → 자력 대피 중일 수 있음
- 연기가 집 안으로 이미 들어왔다면 **일산화탄소 축적** 시작 → 시간이 더 촉박함

### 이 상황에서 LLM이 해야 할 추가 판단

> "조적조로 소방차 대응 가능. 하지만 연기 실내 유입 확인됨 — CO 농도 상승 중일 가능성. 부부 동거이므로 서로 인지했을 가능성 있으나 확인 불가. 마을방송으로 즉시 자력 대피 유도하고, 전화 연락 시도. 감지기 미작동은 유지관리 문제 기록 필요."

## 시나리오 D — FIRE-2026-004 `CRITICAL`

### 상황 서술

새벽 1시 52분. 소록마을에서 가장 위험한 상황.
소록마을 27-2번지 비닐하우스에서 담뱃불로 추정되는 화재가 발생한다.
마을 주민이 불꽃을 목격하고 신고한다.

> *"소록마을인데요, 지금 조 할머니 댁에서 불꽃이 보여요! 비닐하우스 쪽이요. 심야에 혼자 계실 텐데! 치매도 있으셔서 어디 계신지도 몰라요!"*

### 데이터값

| 항목 | 값 |
|------|-----|
| 건물 | BLDG-008 · 소록마을 27-2 · **비닐하우스** · 2005년 |
| 접근로 | **1.5m** — 전 시나리오 중 최협소 |
| 화재감지기 | **없음** |
| 가로등 | **없음** |
| 거주자 | 조옥순(PERSON-015) · **88세** · 여 · 독거 |
| 신체 조건 | **난청 + 치매 + 거동불편** + 고혈압 · mobility_score **0.4** |
| 비상연락처 | 조씨 자녀 · **여수** (30~40분 거리) |
| 기상 | 풍속 6.8m/s · 습도 **18%** · 기온 9.8°C · **건조** |
| 플래시오버 | **약 1.5분** — 전 시나리오 중 최단 |

### 시스템이 해야 하는 판단

1. 비닐하우스 + 습도 18% + 풍속 6.8m/s → 플래시오버 **1.5분** — 시스템 추론 시간(15초)과 합산하면 이미 1.25분밖에 없음
2. 조옥순 88세 + 치매 + 난청 → 경보 인지 불가, 자력 대피 불가
3. 신고자가 "어디 계신지도 몰라요" → **비닐하우스(발화지점)에 있을 가능성 vs 본채에 있을 가능성**
4. 접근로 1.5m → 도보만 가능, 차량 완전 불가

### 현재 시스템의 한계 — 이 시나리오가 가장 극단적

- **1.5분 플래시오버**: 시스템이 15초 추론 후 ARS 발령 → 이장 도착 2분 → **이미 플래시오버 후**
- 비상연락처(여수 자녀)는 30~40분 거리 → 도움 불가
- "어디 계신지 모른다"는 신고 내용 → 수색 위치를 결정해야 하는데 현재 시스템은 판단 불가
- **거주자가 이미 비닐하우스 안에 있다면** 구조 가능성은 신고 시점에 이미 제로에 가까움
- 인접 건물로의 화재 전파 위험 계산 없음

### 이 상황에서 LLM이 해야 할 추가 판단

> "플래시오버 1.5분 — 현재 시각 01:52:07 기준 01:53:37 이미 경과 가능. 비닐하우스 내 생존 가능성 극히 낮음. 즉각 수색 목표를 본채로 전환. 풍속 6.8m/s + 습도 18% → 인접 BLDG-009(경량목구조, 접근로 3.0m)로 3분 내 연소 가능 — 박말순(82세), 정순자(76세) 즉시 대피 경보. 비상연락처 여수 자녀에 상황 통보."

### 데이터값

| 항목 | 값 |

| 비상연락처 | 박씨 자녀 · **서울** (2~3시간 거리) |
| 기상 | 풍속 4.8m/s · 습도 27% · 기온 10.8°C · 건조 |
| 플래시오버 | **약 3분** |

### 시스템이 해야 하는 판단

1. 샌드위치 패널 + 접근로 2.0m → 소방차 진입 불가, 플래시오버 3분
2. 85세 + 치매 + 난청 → 자력 대피 불가, 경보 인지 불가
3. **심장질환** → 구조 과정에서의 처치 방식이 달라져야 함

### 현재 시스템의 한계

- 심장질환 데이터는 있지만 **구조 방식 지시에 반영이 없음**

- 플래시오버 3분 → 마을방송(5초) → 이장(2분) → **도착까지 약 1분 남음**, 매우 촉박

### 이 상황에서 LLM이 해야 할 추가 판단

> "박용철 씨 심장질환 — 구조 시 급격한 자세 변경 금지, 흥분 유발 행동 자제. 신고자가 마을 주민이므로 위치 안내 가이드로 현장 유도 가능. 플래시오버까지 3분, 도보 선발대 출발 기준으로 약 1분 여유. 선발대 최소 2명 — 1명 구조, 1명 유독가스 차단용 입구 확보."

## 결론 — 무엇이 필요한가

현재 시스템이 처리할 수 없는 판단들은 공통적으로 **문맥 의존적**이다.

1. **교차 데이터 추론**: 비상연락처 위치 + 이동시간 → "이 연락은 의미 없다"
2. **우선순위 미세 조정**: mobility_score + 청력 + 치매 조합에 따른 구조 순서
3. **동적 위험 확산**: 풍향 + 건물 간격 → 인접 건물 위험도 실시간 계산
4. **구조 방식 지시**: 기저질환(심장, 뇌졸중 등)에 따른 이동 방법 분기
5. **신고자 활용**: 신고자가 마을 주민인지, 외지인인지에 따라 현장 가이드 역할 부여

이 다섯 가지는 모두 **"if A then B"로 코딩 불가능한 판단**이며, 컨텍스트를 이해하는 LLM 추론 레이어가 있어야 해결된다.
