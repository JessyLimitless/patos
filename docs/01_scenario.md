# PATOS 시나리오 정립
**Predictive Agentic Triage OS — 전남 고흥군 소록마을 심야 화재 대응**
`v4.4 / 2026-04-24`

---

## 1. 배경 및 문제 정의

### 1.1 대상 지역
| 항목 | 내용 |
|------|------|
| 지역 | 전라남도 고흥군 도양읍 소록마을 |
| 좌표 | 34.6101°N, 127.2893°E |
| 특성 | 초고령 농어촌 마을 (65세 이상 비율 67%) |
| 기상 관측소 | 고흥 ASOS 165번 |
| 소방서 ETA | 고흥소방서 → 소록마을 **12분** |

### 1.2 핵심 문제
심야 시간대(00:00~04:00) 농촌 단독주택 화재 발생 시, 표준 소방 대응 체계는 다음 세 가지 구조적 한계에 직면한다.

```
① 인지 지연  : 고령·장애 거주자가 화재 경보를 인지하지 못함 (난청, 치매, 수면)
② 접근 지연  : 협소한 농촌 도로(1.5~2.5m)로 소방차 진입 불가
③ 판단 지연  : 신고 접수 후 우선 구조 대상 선별에 수 분 소요
```

**결과:** 비닐하우스(플래시오버 1.8분), 샌드위치 패널(2.5분) 건물은 소방차 도착(12분) 전에 생존 불가 상태에 도달.

## 2. 시나리오 전제 조건

### 2.1 환경 조건 (심야 화재 공통)
- **시간대:** 02:00~04:00 (주민 수면 중)
- **기상:** 봄철 건조기 (습도 20~40%, 풍속 3~8 m/s)
- **조명:** 가로등 미설치 구간 다수
- **통신:** 119 신고 유일 채널 (마을 자체 경보망 없음)

### 2.2 인프라 전제
- TypeDB 지식 그래프: 주민·건물·자원 데이터 연동 완료
- 기상청 API (Open-Meteo 프록시): 실시간 수신
- 주민 동의 기반 GPS: 동의자 위치 데이터 수신
- 고흥군 건물 대장·주민 등록 DB: 연동 완료

## 5. PATOS 대응 흐름 (4-Step Sequence)

```
[Step 1] 신호 수신 (0~3초)
  119 신고 → STT 변환 → 위치·기상 추출
           ↓
[Step 2] 지식 그래프 매핑 (3~7초)
  건물 DB + 주민 DB + 기상 API → TypeDB 쿼리
           ↓
[Step 3] 온톨로지 추론 (7~15초)
  인지취약성 규칙 + 물리적 고립 규칙 → 우선순위 점수
  화재 전파 물리 모델 → 플래시오버 ETA
           ↓
[Step 4] 자율 대응 발령 (15초~)
  마을방송 · ARS 직발송 · 소방서 MDT 전술정보 동시 송출
```

## 6. 우선순위 점수 산출

```
priority_score = (cognitive_vulnerability × 0.6) + (physical_isolation × 0.4)
```

| 요소 | 만점 | 산출 기준 |
|------|------|-----------|
| cognitive_vulnerability | 100 | 나이 ≥ 80세, 난청, 치매 중 해당 항목 |
| physical_isolation | 100 | 접근로 < 3.0m, 샌드위치 패널·비닐하우스·경량 목구조 |

## 7. 시나리오 성립 조건 및 한계

### 성립 조건
- 주민 DB는 동의서 작성자에 한해 GPS 위치 활용
- 화재 전파 모델은 단층 단독주택 (연면적 85m²) 기준
- 소방차 ETA는 평시 기준 (야간·결빙 등 변수 미적용)

### 한계 및 향후 과제
- 복합 화재(다건 동시 발생) 시나리오 미포함
- 풍향에 따른 연소 방향 예측 미반영
- 실제 소방 API 연동 시 MDT 수신 단말 연계 필요
